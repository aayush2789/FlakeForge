Pipeline

Environment-in-the-loop repair.

Each episode resets a real test repo in Docker, observes pytest stability, applies a structured patch action, and scores with verifiable signals from the environment (not an LLM judge).

01

Observe

Reset the repo, run repeated pytest samples, and record pass rates, errors, and causal hints from the codebase.

02

Reason + patch

The agent emits structured claims and search/replace hunks. The server validates format, applies the patch in the container, and re-runs tests.

03

Verify

Rewards combine stability, compile success, regression checks, and oracle-aligned reasoning — all computed from environment state.

Live run

Real episode from your FF_REPO_PATH.

Calls POST /api/episode/run with repo path, test id, and limits from the server environment. Results below are returned JSON — not scripted demos.
