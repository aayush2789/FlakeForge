How it works

Three-phase causal reasoning.

Every episode follows the same disciplined pipeline: probe the environment, trace the causal boundary, and apply a verified minimal fix.

Phase 01
⚡

Chaos Probe

Inject CPU stress, network delay, or I/O contention to amplify the flake. If the failure rate spikes, the test is infrastructure-sensitive — a strong causal signal.

Phase 02
🎯

Diagnose Boundary

Build a causal graph from test entry to failure point. Isolate the exact file, function, and variable where non-determinism enters the execution path.

Phase 03
🔒

Apply & Verify

Generate a minimal patch using the correct synchronization primitive. Verify with 20 pytest runs — zero regressions required to earn the full reward.
